  • INIT: Pending customer Refund.
  • CONTRACT: The booking has been closed with the supplier
  • REQUEST: The booking has been closed with the supplier without any issue, but still pending the confirmation or the issuance of the same
  • PENDING: The booking is not closed due to there has been an error.
  • RETAINED: A booking is in state retained after creation when we have collected from the user a deposit.
  • HOLD: We are waiting for a user interaction (such as 3DSecure, EBanking, and others third party payments confirmation).
  • REJECTED: After a hold status if the booking is not finally closed.
  • DIDNOTBUY: The booking has been cancelled, this happen usually because of a not recoverable error.
  • AVAILABLE: The product has not been confirmed.
  • SELECTED: The product has been confirmed.
  • EXPIRED: The product expired.
  • REDEEMED: The product has been already used.
  • CANCELLED: The product has been cancelled.
  • COLLECTED. The total amount of the booking has been collected
  • COLLECTION NEEDED. Pending to be collected.
  • NO COLLECTION NEEDED. There is no need to collect (for now).
  • ZERO COLLECTION AMOUNT. 0 shipping and handling.
  • FAILED COLLECTION ATTEMPT. Bookings with errors on payments and finished by agents using PSP.
  • BANK TRANSFER COLLECTED. Transaction confirmed by an agent.
  • MANUALLY COLLECTED. Manually collected by an agent (without using PSP).
  • HIGH. The amount to charge is bigger than the maximum amount defined by the merchant entity.
  • LOCAL ERROR. Error thrown by amadeus API. Invalid reply format.
  • TO RETRY. Deprecated
  • RESPONSE LOST. Error in gateway communication
  • NEEDS INTERACTION: Movement requires user interaction.
  • INIT INTERACTION: Movement inits user interaction process.
  • CANCEL: Movement has been cancelled in payment gateway server.
  • CANCEL ERROR: An error has occured while cancel current movement.
  • DECLINED: Movement has been declined by payment gateway server.
  • PAID: Movement has been paid in payment gateway server
  • PAY ERROR: An error has occured while paying current movement.
  • REFUND ERROR: An error has occured while refunding current movement.
  • REFUNDED: Movement has been refunded in payment gateway server.
  • USE BATCH: Movement is postponed to use batch collection.
  • STATUS: Movement has consulting state of current payment.
  • QUERY ERROR: An error has occured while querying current movement.
  • REQUESTED: Movement cannot be verified
  • CONFIRMED: Flight is confirmed
  • CANCELLED: Flight is cancelled
  • AIRPORT CHANGE: Flight has suffered an airport change
  • DELAYED: Flight has been delayed
  • CHANGED: Flight has suffered a change in the schedule
